# Explore Documentation Directories

When exploring a documentation-type directory (e.g., `docs/`, `agents/`), look for `INDEX.md`.

If present, treat it as the **entry point**. Read it first to understand structure, purpose, and navigation before diving into other files.

## Workflow

1. When entering a documentation-type directory, check for `INDEX.md`
2. If present: read it first to orient yourself
3. Use the index to decide which other files to read and in what order
4. If absent: proceed as usual (no special handling)

## Examples

- `docs/` -> start with `docs/INDEX.md`
- `docs/roadmap/` -> check for `docs/roadmap/INDEX.md`
- `agents/` -> check for `agents/INDEX.md`
